================================================================================
                                1. OVERVIEW
================================================================================

The Elite Gym website is a comprehensive fitness management platform built using
the MERN stack (MongoDB, Express, React, Node.js). It consists of three main
components:

- Frontend: Customer-facing website for browsing, registration, and member access
- Backend: RESTful API server handling all business logic and database operations
- Admin Panel: Management interface for gym administrators

The website features a modern dark theme with neon red (#FF006E) and blue (#00F5FF)
accents, smooth animations, and a fully responsive design.

================================================================================
                          6. PAYMENT SYSTEM
================================================================================

6.1 RAZORPAY INTEGRATION
------------------------
Features:
- Secure payment processing
- Multiple payment methods (cards, UPI, netbanking, wallets)
- Payment verification
- Order management
- Demo mode for testing

How it works:
1. User selects a membership plan
2. Frontend calls /api/payments/create with plan details
3. Backend creates Razorpay order:
   - Calculates amount from plan price
   - Creates order via Razorpay API
   - Returns order ID to frontend
4. Frontend opens Razorpay checkout popup
5. User completes payment
6. Razorpay returns payment details
7. Frontend calls /api/payments/verify with payment details
8. Backend verifies payment signature
9. Payment saved to database
10. User membership activated automatically

6.2 DEMO MODE
-------------
Features:
- Test payments without Razorpay credentials
- Simulates payment flow
- Saves payment data to database
- No actual money transaction

How it works:
- Set DEMO_MODE=true in backend .env
- Set VITE_DEMO_MODE=true in frontend .env
- Payment flow simulates Razorpay checkout
- Payment automatically verified
- Useful for development and testing

6.3 PAYMENT VERIFICATION
------------------------
How it works:
- Razorpay provides payment signature
- Backend verifies signature using Razorpay secret key
- Signature verification ensures payment authenticity
- Only verified payments activate membership
- Failed verifications marked as failed

================================================================================
                 7. AUTHENTICATION & AUTHORIZATION
================================================================================

7.1 USER AUTHENTICATION
-----------------------
Features:
- User registration with email and password
- Login with credentials
- JWT token-based authentication
- Password hashing with bcrypt
- Session persistence
- Protected routes

How it works:
- Registration: User provides email, password, name
- Password hashed using bcrypt before storage
- Login: Credentials verified against database
- JWT token generated on successful login
- Token stored in HTTP-only cookie and localStorage
- Token included in API requests via Authorization header
- Token expiration: 7 days (configurable)
- Protected routes check for valid token

7.2 ADMIN AUTHENTICATION
------------------------
Features:
- Separate admin login system
- Super admin and regular admin roles
- Role-based access control
- Admin registration (super admin only)

How it works:
- Admin login via /api/admin/auth/login
- Separate Admin model in database
- Super admin can create other admins
- Roles determine access level:
  * super_admin: Full access to all features
  * admin: Limited access (cannot manage other admins)
- JWT tokens for admin sessions
- Admin panel uses separate authentication

7.3 ROLE-BASED ACCESS CONTROL
------------------------------
Roles:
- visitor: Unregistered user, limited access
- member: Registered user with active membership
- admin: Gym staff with management access
- super_admin: Full system access

How it works:
- User roles stored in database
- Middleware checks role before allowing access
- Protected routes specify allowed roles
- Frontend shows/hides features based on role
- API endpoints verify role before processing requests

7.4 PROTECTED ROUTES
--------------------
Features:
- Route protection based on authentication
- Role-based route access
- Automatic redirect to login
- Session validation

How it works:
- ProtectedRoute component wraps protected pages
- Checks authentication status
- Verifies user role matches required roles
- Redirects to login if not authenticated
- Preserves intended destination for redirect after login

Project Structure:
